# Database Design Summary: CountIt [Jewellery]

## 1. Master DB (Shared)
- `Tenants`
  - tenant_id (PK)
  - name
  - subscription_plan
  - created_at
- `Users`
  - user_id (PK)
  - tenant_id (FK)
  - email
  - role
  - permissions (JSON or separate table)
- `Subscriptions`
  - subscription_id (PK)
  - tenant_id (FK)
  - plan_details
  - status

## 2. Tenant DB (Per Tenant)
- `Branches`
  - branch_id (PK)
  - name
  - currency
  - settings
- `Products`
  - product_id (PK)
  - sku (unique)
  - name
  - base_unit
  - tax_category_id (FK)
  - is_discountable
  - allow_price_change
- `ProductUnits`
  - unit_id (PK)
  - product_id (FK)
  - unit_name
  - conversion_factor
- `ProductVendorRates`
  - rate_id (PK)
  - product_id (FK)
  - vendor_id (FK)
  - unit
  - rate
- `ProductCustomerTypeRates`
  - rate_id (PK)
  - product_id (FK)
  - customer_type_id (FK)
  - rate
- `Batches`
  - batch_id (PK)
  - product_id (FK)
  - branch_id (FK)
  - batch_no
  - quantity
- `InventoryTransactions`
  - txn_id (PK)
  - product_id (FK)
  - batch_id (FK)
  - branch_id (FK)
  - type (purchase/sales/transfer/adjustment/consumption)
  - quantity
  - date
- `TaxCategories`
  - tax_category_id (PK)
  - name
  - rate
- `Customers`, `Suppliers`, `Employees`
  - Each with ledger_id (FK) to `Ledgers`
- `Ledgers`
  - ledger_id (PK)
  - name
  - type (customer/supplier/employee)

## 3. Key Relationships
- Tenant → Branches → Products → Batches → InventoryTransactions
- Products ↔ TaxCategories, Discounts, VendorRates, CustomerTypeRates
- Customers/Suppliers/Employees ↔ Ledgers
