# QA Plan: CountIt [Jewellery]

## 1. Testing Strategy
- **Unit Testing**: Backend services, business logic (tax, discounts, inventory)
- **Integration Testing**: API endpoints, DB interactions, multi-tenant flows
- **System Testing**: End-to-end flows (tenant onboarding → product setup → POS → payments)
- **UAT**: Client-led testing with real-world scenarios

## 2. Test Environments
- Dev, Staging, Production
- Tenant DB provisioning process in each environment

## 3. Test Data Requirements
- Sample tenants, branches, products, batches, tax rules, customers, suppliers

## 4. Test Scenarios (Examples)
- Tenant registration and DB creation
- Product creation with multiple units and attributes
- Purchase with batch and VAT
- Sales with batch selection, tax, discounts
- Stock transfer between outlets with approval
- Customer payment with TDS and additional headings

## 5. Definition of Done (DoD)
- Code reviewed
- Unit/integration tests passing
- Tested on staging
- Documentation updated
