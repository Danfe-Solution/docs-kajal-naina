# API Specification Summary: CountIt [Jewellery]

## 1. Tenant & RBAC
- `POST /api/tenants/register`
  - Request: { email, businessName, ... }
  - Response: { tenantId, userId, message }
- `POST /api/tenants/{tenantId}/admins/invite`
  - Request: { email, role }
  - Response: { invitationId, status }
- `GET /api/users/me`
  - Response: { userId, tenantId, role, permissions }

## 2. Product & Inventory
- `POST /api/products`
  - Request: { sku, name, units[], attributes[], taxCategoryId, ... }
  - Response: { productId, sku, ... }
- `GET /api/products?tenantId=...&branchId=...`
  - Response: List of products with rates and stock info
- `POST /api/inventory/purchase`
  - Request: { productId, batchNo, quantity, branchId, isVat, claimableVatPercent, ... }
  - Response: { txnId, batchId, stockBalance }
- `POST /api/inventory/sales`
  - Request: { items: [{ productId, batchId, quantity }], tax, discounts, paymentMethod, ... }
  - Response: { saleId, receiptUrl, stockUpdates }

## 3. POS & Sales
- `POST /api/pos/sales`
  - Request: { branchId, terminalId, cartItems, paymentDetails }
  - Response: { saleId, receipt }
- `GET /api/pos/terminals?branchId=...`
  - Response: List of POS terminals for the branch

## 4. Payments
- `POST /api/payments/customer`
  - Request: { customerId, saleId, tds, additionalHeadings[] }
  - Response: { paymentId, ledgerEntries }
- `POST /api/payments/supplier`
  - Request: { supplierId, purchaseId, tds, additionalHeadings[] }
  - Response: { paymentId, ledgerEntries }
